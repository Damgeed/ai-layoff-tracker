1. Core Fields

Every entry in the database must include all core fields. These fields identify the company, the event, and the magnitude of the workforce reduction.

Field Name Type Required Description Example Constraints
id string Yes Unique identifier for the entry, combining company slug and date (YYYY-MM). klarna-2024-08 Pattern: {slug}-{YYYY}-{MM}. Must be unique across all entries.
company string Yes Full legal or public-facing company name. Klarna Must match a known public company. Case-sensitive.
slug string Yes URL-safe company identifier. Used for routing and lookups. klarna Lowercase alphanumeric with hyphens. Must match /company/{slug}/.
date string Yes Primary date of the workforce reduction announcement or event. 2024-08-27 ISO 8601 date format (YYYY-MM-DD).
country string Yes Country where the affected workforce is primarily located. Sweden Must be a valid value from the countries list.
industry string Yes Primary industry classification for the company. Fintech Must be a valid value from the industries list.
jobs_lost number Yes Total number of jobs affected by the workforce reduction. 2000 Integer. Must be ≥ 1. For hiring freezes, this is the estimated roles affected.
workforce_before number Yes Company workforce headcount before the reduction. 5000 Integer. Must be ≥ jobs_lost. Used to calculate impact_percent.
workforce_after number Yes Company workforce headcount after the reduction. 3000 Integer. Should equal workforce_before - jobs_lost for layoffs. May differ for hiring freezes.
impact_percent number Yes Percentage of total workforce affected, rounded to nearest integer. 40 0–100. Calculated as round((jobs_lost / workforce_before) × 100).

2. Classification

These fields record the AI-attribution tier, verification status, and confidence assessment for each entry. They are the evidentiary backbone of the database.

Field Name Type Required Description Example Constraints
classification string Yes The classification tier indicating how directly AI was cited as a factor. DIRECT_AI_REPLACEMENT Must be one of: DIRECT_AI_REPLACEMENT, AI_DRIVEN_RESTRUCTURING, AI_REALLOCATION, MARKET_DISRUPTION.
confidence_score number Yes Evidence strength score reflecting how directly and unambiguously AI is linked to the workforce reduction. 95 Integer 0–100. Minimum 50 for publication. See Classification Reference for tier thresholds.
verified boolean Yes Whether the entry has passed verification review against primary sources. true true for published entries. false indicates the entry is in a pre-verification state. See Status Lifecycle.
evidence array Yes Array of evidence objects documenting the types of source material supporting this entry. [{"type":"CEO_STATEMENT","description":"..."}] At least one evidence object required. Each object must have type (valid evidence type) and description (string).

3. Source

The source object captures the primary reference used to validate the entry. Every published entry must have a complete source object with an accessible URL and retrieval metadata.

Field Name Type Required Description Example Constraints
source.title string Yes Title of the news article, press release, filing, or source document. BBC News Should reflect the original publication's headline or a concise descriptive title.
source.url string Yes Direct URL to the original source document. https://www.bbc.com/news/articles/c4gpqp4v4q2o Must be a valid, publicly accessible HTTPS URL. Domain should be on the trusted-source whitelist.
source.publisher string Yes The organization or entity that published the source. BBC News Should be a recognized news organization, company, or regulatory body.
source.published_date string Yes Publication date of the source document. 2024-08-27 ISO 8601 date format (YYYY-MM-DD).
source.retrieved_date string Yes Date the source was last accessed and verified by the maintainer. 2026-06-13 ISO 8601 date format (YYYY-MM-DD).
source.archive_url string Yes Permalink to an archived copy of the source (e.g., Internet Archive Wayback Machine). https://web.archive.org/web/*/https://www.bbc.com/… Must be a valid URL. Ensures long-term citability even if the original page changes.

4. Content

These fields provide the human-readable content, key quotes, and metadata tags that describe the workforce reduction in context.

Field Name Type Required Description Example Constraints
summary string Yes A factual, editorial-free summary of the workforce reduction and its AI attribution. Klarna CEO stated the company reduced its workforce from 5,000 to 3,000 by leveraging AI… Should not contain opinion or speculation. Based solely on verified source material.
ceo_quote string No Direct verbatim quote from a company executive explicitly linking AI to the workforce reduction. We can do the same amount of work with 2,000 employees that we previously did with 5,000 people. Only included when a direct, attributable quote exists. May be null or empty string.
tags array No Array of lowercase keyword tags for filtering and discovery. ["customer-support", "generative-ai", "chatbots", "automation"] Lowercase, hyphenated strings. No spaces. Common tags: automation, generative-ai, restructuring, chatbots, ai-pivot, etc.

5. Status Lifecycle

Every entry moves through a defined lifecycle from initial submission to final state. The verified boolean field reflects the current stage.

PENDING
→
VERIFIED
→
DISPUTED
→
RETRACTED
Status verified Description Transition Rules
PENDING false Entry has been submitted or identified but has not yet undergone full verification. Not publicly visible. Default state for new entries. Moves to VERIFIED upon passing verification.
VERIFIED true Entry has passed verification: primary source confirmed, cross-referenced, classification assigned, and confidence scored. Publicly visible. Can be moved to DISPUTED if a credible challenge is raised. Cannot revert to PENDING.
DISPUTED true A credible challenge has been raised against the entry's classification, data, or evidence. Under re-review. Remains publicly visible with a dispute notice. Can return to VERIFIED if challenge is resolved in favor of original entry, or move to RETRACTED if challenge is upheld.
RETRACTED false Entry has been retracted due to discredited evidence, factual error, or withdrawal by the source. Not publicly visible. Retained for audit trail. Final state. Once retracted, an entry cannot be re-published; a new entry ID must be created if warranted.

Note: The status is currently encoded via the verified boolean and contextual metadata. A future schema version may introduce an explicit status enum field for clarity.

6. Classification Reference

Every entry is assigned to one of four classification tiers based on how directly and explicitly AI is cited as a factor in the workforce reduction.

Tier Classification Value Definition Confidence Threshold
Tier 1 DIRECT_AI_REPLACEMENT Company explicitly states AI systems replaced specific human work, resulting in workforce reduction. 80+
Tier 2 AI_DRIVEN_RESTRUCTURING Company cites AI efficiency or automation as a major factor in a restructuring that included layoffs. 70+
Tier 3 AI_REALLOCATION Jobs cut while the company simultaneously announced increased investment in AI, with resources redirected toward AI priorities. 70+
Tier 4 MARKET_DISRUPTION Company lost revenue or market position because customers switched to AI-powered alternatives, leading to workforce reductions. 80+

7. Evidence Types

Each entry's evidence array contains one or more objects with a type field. The following seven evidence types are valid:

Type Value Definition Typical Source Example
CEO_STATEMENT Public statement by the company's CEO or equivalent executive directly addressing AI's role in workforce changes. Earnings call, media interview, company blog post by CEO
COMPANY_STATEMENT Official company press release, blog post, or spokesperson statement. Press release, official company blog, spokesperson quote to media
EARNINGS_REPORT Quarterly or annual earnings filing, investor presentation, or shareholder letter. Quarterly earnings call transcript, investor deck, annual report
SEC_FILING Filing with the U.S. Securities and Exchange Commission. 10-K, 10-Q, 8-K, S-1, DEF 14A
MEDIA_REPORT Verified reporting by a recognized news organization with named sources. Reuters, Bloomberg, BBC, CNBC, The Verge, Ars Technica
REGULATORY_FILING Filing with government agencies outside the U.S., including workforce reduction notifications. WARN Act notice (U.S. states), EU collective redundancy notification, country-specific labor filings
TRAFFIC_ANALYSIS Third-party web/app traffic data showing measurable decline linked to AI competition. Used primarily for Tier 4 entries. SimilarWeb, Sensor Tower, App Annie, company-reported user metrics
