Citation Guide

How to cite the AI Layoff Tracker in academic papers, news articles, and research publications. The dataset is CC0 — attribution is appreciated but not legally required.

General Guidance

The AI Layoff Tracker dataset is dedicated to the public domain under CC0 1.0 Universal. You may use, modify, and redistribute the data for any purpose. While attribution is not legally required, we strongly recommend citing the dataset in your work. Citing the dataset:

  • Allows readers to verify your claims against the source data.
  • Credits the researchers who verified each entry.
  • Enables reproducibility — readers can retrieve the exact version you used.

Always include the version number and retrieval date when citing the dataset. The dataset is updated on a rolling basis and version numbers anchor your citation to a specific snapshot.

APA Format (7th Edition)

Use the following format to cite the full dataset in APA style:

AI Layoff Tracker. (2026). AI Layoff Tracker database (Version 1.0) [Data set]. Retrieved June 13, 2026, from https://damgeed.github.io/ai-layoff-tracker/api/entries.json

In-text citation: (AI Layoff Tracker, 2026)

MLA Format (9th Edition)

Use the following format to cite the full dataset in MLA style:

"AI Layoff Tracker Database." AI Layoff Tracker, version 1.0, 13 June 2026, damgeed.github.io/ai-layoff-tracker/api/entries.json. Accessed 13 June 2026.

In-text citation: ("AI Layoff Tracker Database")

Chicago Manual of Style (17th Edition, Notes & Bibliography)

Note (first citation):

1. "AI Layoff Tracker Database," AI Layoff Tracker, version 1.0 (2026), accessed June 13, 2026, https://damgeed.github.io/ai-layoff-tracker/api/entries.json.

Bibliography:

AI Layoff Tracker. "AI Layoff Tracker Database." Version 1.0. Accessed June 13, 2026. https://damgeed.github.io/ai-layoff-tracker/api/entries.json.

How to Cite Individual Entries

To cite a specific entry rather than the full dataset, reference the entry's dedicated page. Each entry has a stable URL that will not change. For example, to cite the Klarna entry:

APA — Individual Entry:

AI Layoff Tracker. (2026). Klarna (Entry ID: klarna-2024-08). In AI Layoff Tracker database (Version 1.0). Retrieved June 13, 2026, from https://damgeed.github.io/ai-layoff-tracker/entry/klarna-2024-08/

MLA — Individual Entry:

"Klarna (Entry ID: klarna-2024-08)." AI Layoff Tracker, version 1.0, 2026, damgeed.github.io/ai-layoff-tracker/entry/klarna-2024-08/. Accessed 13 June 2026.

Replace klarna-2024-08 with the entry ID of the specific entry you are citing. Every entry page includes the full citation-ready metadata.

Dataset Versioning

The AI Layoff Tracker uses semantic versioning for the dataset. The current version is v1.0 (initial release, 27 entries, 80,794 documented jobs, June 13, 2026). When citing the dataset, always note the version number and your retrieval date. This ensures your citation points to the exact snapshot you analyzed, even if entries are added or modified later.

Future versions will follow this scheme:

  • Patch versions (v1.0.1, v1.0.2…): Corrections to existing entries.
  • Minor versions (v1.1, v1.2…): New entries added, no breaking schema changes.
  • Major versions (v2.0, v3.0…): Schema changes, reclassifications, or substantial restructuring.

All dataset releases are documented on the Dataset Versions page. All entry-level changes are logged in the Changelog and the Entry History audit trail.
